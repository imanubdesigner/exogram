# Environment Variables

Exhaustive reference for all variables consumed by the project.
Backend variables are read with `python-decouple` from `backend/.env`.
Frontend variables are passed as build arguments (`VITE_*`).
Root compose variables are read from `.env` in the project directory.

---

## How the file hierarchy works

```
exogram/
├── .env                ← Root compose variables (APP_DOMAIN, ACME_EMAIL, Postal, Sentry)
└── backend/
    └── .env            ← Django/Celery variables (SECRET_KEY, DB, Redis, JWT, email, etc.)
```

In production, both files are generated by the deploy workflow from GitHub secrets.
In development, you create them manually (see [Local Development](./local-development.md)).

---

## Root compose — `.env`

Used by `docker-compose.yml` and `docker-compose.prod.yml`.

| Variable | Required | Example | Description |
|---|---|---|---|
| `APP_DOMAIN` | Yes (prod) | `exogram.app` | Main domain. Caddy uses it for TLS and routing. |
| `ACME_EMAIL` | Yes (prod) | `admin@exogram.app` | Email for Let's Encrypt. Receives expiration notices. |
| `SENTRY_DSN` | No | `https://abc@sentry.io/123` | Sentry DSN. Empty = Sentry disabled. |
| `REDIS_URL` | No | `redis://redis:6379/0` | Redis URL. Internal compose default. |
| `VITE_DONATION_ALIAS` | No | `@alias` | Donation alias displayed in the UI. |
| `POSTAL_DB_ROOT_PASSWORD` | Only if using Postal | `s3cr3t` | MariaDB root password for Postal. |
| `POSTAL_DB_PASSWORD` | Only if using Postal | `s3cr3t` | Password for the `postal` user in MariaDB. |
| `POSTAL_RAILS_SECRET_KEY` | Only if using Postal | long string | Postal's internal Rails secret key. |

## Backend — `backend/.env`

Used by Django, Celery, and Gunicorn.

### Security and Django

| Variable | Required | Example | Description |
|---|---|---|---|
| `SECRET_KEY` | **Yes** | 50+ char string | Django master key. Used to sign sessions, CSRF tokens, and invitation HMACs. Never reuse between environments. Generate with `python -c "import secrets; print(secrets.token_urlsafe(50))"`. |
| `DEBUG` | No | `False` | `True` only in development. Must be `False` in production. |
| `ALLOWED_HOSTS` | **Yes** | `exogram.app,www.exogram.app` | Hosts accepted by Django. Comma-separated. Include the domain and its variants. |
| `DJANGO_SETTINGS_MODULE` | No | `exogram.settings` | Settings module to use. Default: `exogram.settings`. |
| `ADMIN_URL` | No | `gestion-secreta/` | Django admin path. Obfuscate to avoid enumeration. Default: `admin/`. |
| `FORCE_HTTPS` | No | `True` | Enables HSTS, SSL redirect, and Secure cookies. Only in production with TLS. |

### Database

Can be provided as `DATABASE_URL` (compact form) or as individual variables.
If both are present, `DATABASE_URL` takes precedence.

| Variable | Required | Example | Description |
|---|---|---|---|
| `DATABASE_URL` | No* | `postgres://user:pass@db:5432/exogram` | Full connection URL. |
| `POSTGRES_DB` | No* | `exogram` | Database name. |
| `POSTGRES_USER` | No* | `exogram` | PostgreSQL user. |
| `POSTGRES_PASSWORD` | No* | `s3cr3t` | PostgreSQL password. |
| `DB_HOST` | No | `db` | PostgreSQL host. Default: `db` (Docker service name). |
| `DB_PORT` | No | `5432` | PostgreSQL port. Default: `5432`. |

*One of the two forms is required.

### Redis and Celery

| Variable | Required | Example | Description |
|---|---|---|---|
| `CELERY_BROKER_URL` | No | `redis://redis:6379/0` | Celery broker URL. Default: `redis://redis:6379/0`. |
| `CELERY_RESULT_BACKEND` | No | `redis://redis:6379/0` | Celery result backend. Default: same as broker. |
| `REDIS_URL` | No | `redis://redis:6379/0` | Used by the Celery Beat heartbeat and health check. |

### JWT and authentication cookies

| Variable | Required | Example | Description |
|---|---|---|---|
| `JWT_ACCESS_TOKEN_MINUTES` | No | `20` | Access token lifetime in minutes. Default: `20`. |
| `JWT_REFRESH_TOKEN_DAYS` | No | `7` | Refresh token lifetime in days. Default: `7`. |
| `JWT_ACCESS_COOKIE_NAME` | No | `exo_access` | Access token cookie name. Default: `exo_access`. |
| `JWT_REFRESH_COOKIE_NAME` | No | `exo_refresh` | Refresh token cookie name. Default: `exo_refresh`. |
| `JWT_ACCESS_COOKIE_PATH` | No | `/api/` | Access cookie path. Default: `/api/`. |
| `JWT_REFRESH_COOKIE_PATH` | No | `/api/auth/` | Refresh cookie path. Restrict to minimum. Default: `/api/auth/`. |
| `JWT_COOKIE_SECURE` | No | `True` | Marks cookies as Secure (HTTPS only). Inherited from `FORCE_HTTPS` if not set. |
| `JWT_COOKIE_SAMESITE` | No | `Lax` | SameSite policy for JWT cookies. Default: `Lax`. |
| `JWT_COOKIE_DOMAIN` | No | `.exogram.app` | Cookie domain. Empty = same domain as the request. |

### CORS and CSRF

| Variable | Required | Example | Description |
|---|---|---|---|
| `CORS_ALLOWED_ORIGINS` | No | `https://exogram.app` | Allowed CORS origins. Comma-separated. Default: `localhost:5173`. |
| `CSRF_TRUSTED_ORIGINS` | No | `https://exogram.app` | Trusted origins for CSRF. Comma-separated. |

### Email (Postal / SMTP)

| Variable | Required | Example | Description |
|---|---|---|---|
| `EMAIL_HOST` | No | `postal` | SMTP host. Default: `postal` (Docker service name). |
| `EMAIL_PORT` | No | `587` | SMTP port. Default: `587`. |
| `EMAIL_HOST_USER` | No | `noreply@exogram.app` | SMTP user. |
| `EMAIL_HOST_PASSWORD` | No | `s3cr3t` | SMTP password. |
| `EMAIL_USE_TLS` | No | `True` | Use STARTTLS. Default in settings: `True`. |
| `EMAIL_USE_SSL` | No | `False` | Use direct SSL (port 465). Incompatible with `EMAIL_USE_TLS`. |
| `DEFAULT_FROM_EMAIL` | No | `Exogram <noreply@exogram.app>` | Default sender for all emails. |
| `EMAIL_BACKEND` | No | `django.core.mail.backends.smtp.EmailBackend` | Email backend. Change to `locmem` or `console` for local debug. |

### URLs and frontend

| Variable | Required | Example | Description |
|---|---|---|---|
| `FRONTEND_BASE_URL` | No | `https://exogram.app` | Frontend base URL. Used to build links in invitation and password reset emails. Default: `http://localhost:5173`. |
| `OPENLIBRARY_API_URL` | No | `https://openlibrary.org/api` | OpenLibrary base URL for metadata enrichment. Default: the value shown. |
| `PASSWORD_RESET_TOKEN_TTL_HOURS` | No | `2` | Password reset token validity in hours. Default: `2`. |

### Observability

| Variable | Required | Example | Description |
|---|---|---|---|
| `SENTRY_DSN` | No | `https://abc@sentry.io/123` | Sentry DSN for the backend. Empty = disabled. |

## Frontend — build variables (`VITE_*`)

Passed at build time as `VITE_API_URL=https://... npm run build`.
The deploy workflow injects them automatically from GitHub secrets.

| Variable | Required | Example | Description |
|---|---|---|---|
| `VITE_API_URL` | No | `https://exogram.app` | Backend base URL. The frontend appends `/api`. Empty = `http://localhost:8000/api`. |
| `VITE_DONATION_ALIAS` | No | `@alias` | Donation alias visible in the UI. Optional. |
