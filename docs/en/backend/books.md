# App: books

The core of Exogram. Manages each user's personal library: books, highlights,
notes, import from external sources, semantic search, and Goodreads sync.

---

## Models

### `Author`

Author of a book.
- `name` (effectively unique by normalization)
- `openlibrary_id` — ID in OpenLibrary, for metadata enrichment.

### `Book`

Book with metadata.
- `title`, `isbn` (unique, nullable)
- `authors` — ManyToMany with `Author`
- `cover_image` — uploaded or imported cover
- `openlibrary_id`, `average_rating`, `publish_year`, `genre`
- Metadata is asynchronously enriched from OpenLibrary via a Celery task.

### `Highlight`

The central object of the application. A quote or marked passage from a book.

- `user` → FK to `Profile`
- `book` → FK to `Book`
- `content` — highlight text
- `note` — user's personal note about the highlight (optional)
- `location` — position in the book ("Loc 450-455", "Page 123")
- `visibility`: `private` | `unlisted` | `public`
- `embedding` — `VectorField(dimensions=384)`. Asynchronously generated by Celery using the ONNX model. Null until the worker processes it.
- `is_favorite`
- HNSW index on the `embedding` field for efficient vector search.

## Highlight import

### Kindle

Custom parser (`books/parsers/kindle_parser.py`) that processes the `My Clippings.txt` file
generated by the Kindle. Extracts books, authors, highlights, and notes by splitting on the
`==========` delimiter. Tolerant of malformed rows.

### Goodreads

Two mechanisms:
1. **Goodreads export CSV:** manual import of the file Goodreads generates in its export panel.
2. **Goodreads RSS feed:** periodic sync via Celery that consumes the user's public "currently reading" / "read" feed.

The Goodreads scraper (`goodreads_reading_scraper.py`) parses external HTML, so
it is tolerant of structural changes (errors in individual rows do not abort the import).

### Import endpoint

`POST /api/books/import/` — accepts Kindle or Goodreads CSV files.
Processes synchronously (parsing) and dispatches embedding tasks asynchronously.

## Embeddings and semantic search

### Embedding generation

After importing highlights, Celery tasks are dispatched to generate embeddings:

- **`generate_highlight_embedding(highlight_id)`**: generates the embedding for a single highlight.
- **`batch_generate_embeddings(highlight_ids)`**: processes multiple highlights in sub-batches of 16. Only processes highlights without an embedding (`embedding__isnull=True`), making it idempotent.
- If the ONNX model is not available, the task returns a warning and does not fail. Highlights remain without an embedding until the model is available.

### Semantic search

`POST /api/discovery/search/` with `{query: "search text", scope: "mine"|"network", limit: N}`

1. The search text is converted to an embedding using the same ONNX model.
2. A cosine similarity search is executed in pgvector against the highlights in the chosen scope.
3. The HNSW index on `Highlight.embedding` speeds up the search.
4. Configurable maximum: 50 results (`max_value=50`).

## Celery tasks

### `enrich_book_metadata(book_id)`

Queries the OpenLibrary API by ISBN. If the book is found, updates `publish_year` and `genre`.
Uses retry with exponential backoff on network errors (max 3 attempts).

### `generate_highlight_embedding(highlight_id)`

See the embeddings section.

### `batch_generate_embeddings(highlight_ids)`

See the embeddings section.

### `promote_trust_levels_task()`

Bulk-promotes users with >= 30 days of seniority from `depth=0` to `depth=1`.
Runs daily via Celery Beat.

### `beat_heartbeat()`

Writes a timestamp to Redis every 60 seconds. The beat container's healthcheck validates this.

### Goodreads sync tasks

- `sync_goodreads_rss(profile_id)`: syncs a user's RSS feed.
- `sync_goodreads_reading(profile_id)`: syncs the "currently reading" status.
- `sync_all_goodreads_feeds()`: dispatches sync for all users with Goodreads enabled.

## Highlight visibility

| Value | Behavior |
|---|---|
| `private` | Only visible to the owner |
| `unlisted` | Does not appear in public listings but accessible via direct URL |
| `public` | Visible on the public profile and in the community feed |

---

## Public notes

Highlights with `visibility=public` and a non-empty `note` are accessible as "public notes".
`GET /api/books/public-notes/<nickname>/` returns the public notes of a user.

---

## Export

`GET /api/books/export/` — exports all highlights for the authenticated user in
CSV or JSON format (based on the `Accept` header or `format` parameter).

---

## Main endpoints

| Method | URL | Description |
|---|---|---|
| GET | `/api/books/` | List user's books |
| POST | `/api/books/import/` | Import highlights (Kindle / Goodreads CSV) |
| GET/PATCH/DELETE | `/api/books/<id>/` | Book detail, edit, and delete |
| GET | `/api/books/<id>/highlights/` | Highlights for a book |
| PATCH | `/api/highlights/<id>/` | Update highlight (note, visibility, favorite) |
| GET | `/api/books/export/` | Export highlights |
| GET | `/api/books/public-notes/<nickname>/` | Public notes for a user |
| POST | `/api/discovery/search/` | Semantic similarity search |
